Colores y textos

REGLAS CSS

Una regla CSS es un conjunto de declaraciones que aplican a un selector.

Un selector puede ser un o varios elementos HTML o una selección de elementos que tengan un atributo concreto. Ejemplo: “h1”

Una declaración se compone de una propiedad CSS y su valor. Ejemplo: “color: red”

En un documento CSS pueden existir múltiples reglas de CSS.

h1 { color:red; margin: 20px; }

Integración CSS

Existen 3 formas distintas de usar CSS en el HTML:

  1. Estilos inline

    Las declaraciones se introducen en el mismo fichero HTML dentro de las etiquetas mediante el atributo style. Es la menos recomendada ya que hay que indicar el estilo para cada uno de los elementos.

  2. Estilos dentro de la etiqueta <head>

    Mediante la etiqueta style, podemos definir las reglas CSS para todo el documento en la cabecera del documento HTML

  3. Documento CSS enlazado

    Podemos enlazar uno o varios documentos HTML con un documento CSS externo que será válido para todos. Es la opción más recomendada ya que cualquier cambio de estilo aplicará a todos los HTML enlazados.

<h1 style="color:red;font-size:12px;">text</h1> <head> ... <style> h1 { color:red; margin: 20px; } </style> <head>

Colores

  • color

    Define el color que queremos para el texto

  • background-color

    Define el color que queremos como fondo

Existen varias maneras de definir un color en CSS. Las más comunes son:

  • por nombre

    existen una serie de colores básicos con un nombre asignado. Ej: white, black, green, blue, red, orange, …

  • por notación hexadecimal

    es un conjunto de 6 carácteres que van del 0 al 9 y de la A a la F con el símbolo de “hashtag” delante. Ej: #000000 , #238A6C, #FFFFFF

  • por notación RGBa

    es un conjunto de 4 números. Los tres primeros pueden ir de 0 a 255 y indican la cantidad de rojo, verde o azul que va a tener el color. El último número es la opacidad y puede ir de 0 a 1, siendo 0 transparente y 1 opaco.

h1 { color:red; background-color: green; } /* color rojo en las distintas notaciones */ color: red; color: #FF0000; color: rgba(255,0,0,1);

Si quieres obtener un color, busca en google “color picker” y te aparecerá una paleta para obtener el valor en cualquiera de las notaciones que hemos visto

Borders, Margins, Paddings y Modelo de caja

Borders

Podemos definir un marco para los elementos HTML con la propiedad “border”, que necesitará tres valores:

  • tamaño del border, habitualmente en “px”
  • estilo del border (solid, dashed, dotted, double, groove, …)
  • color del borde

Se pueden definir los valores de los borders por separado a través de las propiedades border-top, border-right, border-bottom, border-left

h1 { border: 5px solid red; border-top: 10px double green; }

Margins y paddings

Podemos definir los márgenes para los elementos HTML mediante dos propiedades:

  • margin, establece la separación entre el border del elemento y los elementos colindantes.
  • padding establece la separación entre el border y los elementos interiores

Se pueden definir los valores de los margin y padding por separado a través de las propiedades:

  • margin-top, margin-right, margin-bottom, margin-left
  • padding-top, padding-right, padding-bottom, padding-left
h1 { margin: 5px; padding: 5px; margin-top: 10px; padding-top: 10px; }

Modelo de caja

Al conjunto de margin + border + padding se le llama el modelo de caja (box model).

Unidades

Para definir separaciones o tamaños es habitual utilizar las siguientes unidades:

  • px: establece el tamaño mediante pixels. Debemos conocer que una pantalla de móvil tiene entre 320px y 375px de ancho por 568px y 667px de alto, y los ordenadores suelen tener entre 1280x y 1920px de ancho por 1080px de alto.
  • em: se trata de un tamaño relativo basado en el ancho del carácter “m”. Si definimos un tamaño de 2em, estamos diciendo que la fuente es el doble de lo esperado. Si definimos 0.5em, será la mitad de lo esperado.
  • rem: se trata de un tamaño relativo basado en el ancho del carácter “m”. Si definimos un tamaño de 2em, estamos diciendo que la fuente es el doble de lo esperado. Si definimos 0.5em, será la mitad de lo esperado.

Selectores CSS

Introducción

Son los patrones para seleccionar elementos en nuestro HTML.

Lo más habitual es usar nombres de elementos, clases (class) o identificadores (id)

<div class="container" id="intro">Nuclio School<div> div { color: red; } div.container { color: green; } div#intro { color: blue; }

Descendientes

Dependiendo del patrón, podremos seleccionar solo los descendientes de un elemento concreto.

Las estructuras más habituales son:

  • selector1 selector2: todos los selector2 que se encuentran dentro de selector1
  • selector1 > selector2: todos los selector2 que son hijos directos de selector1
  • selector1 ~ selector2: Cualquier selector2 que se encuentra precedido de selector1
  • selector1 + selector2: Solo el primer selector2 que se encuentra imediatamente después de selector1
div ul { color: red; } div > ul { color: green; } div ~ ul { color: blue; } div + ul { color: orange; }

Selectores avanzados

  • * : todos los elementos
  • selector1 , selector2: todos los selector1 y selector2
  • [atributo]: Todos los elementos que tengan este atributo
  • [atributo=valor]: Todos los elementos que tengan este atributo con este valor
  • [atributo~=valor]: Todos los elementos que tengan en su valor la palabra del valor indicado
  • [atributo^=valor]: Todos los elementos que tengan este atributo y que su valor empieze por el valor indicado
  • [atributo$=valor]: Todos los elementos que tengan este atributo y que su valor acabe por el valor indicado
  • [atributo*=valor]: Todos los elementos que tengan este atributo y que su valor contenga el valor indicado

Pseudo clases y pseudo elementos

Pseudo clases

Las pseudo-clases sirven para seleccionar elementos según su estado. Van precedidos del carácter “:”

  • :checked : todos los checkbox marcados
  • :disabled : todos los inputs desactivados
  • :enabled : todos los inputs activos
  • :focus : el input que tiene el foco
  • :required : todos los que son requerido
  • :invalid : todos los inputs que tiene un valor inválido
  • :valid : todos los inputs que tiene un valor válido
  • :hover : todos los elementos que el ratón pase por encima

Existen pseudo-clases para seleccionar un hijo concreto:

  • :first-child : El primer elemento del grupo
  • :last-child : El último elemento del grupo
  • :nth-child(n) : El elemento que responde a la fórmula entre paréntesis

Pseudo elementos

Son elementos que no existen pero los podemos seleccionar. Van precedidos de los caracteres: “::” y los más habituales son:

  • ::before: el pseudo-elemento que precede al elemento
  • ::after: el pseudo-elemento contiguo al elemento
  • ::first-letter: la primera letra del elemento
  • ::selection: la selección del usuario

Tipografía

Fuentes tipográficas

Existen varias propiedades relacionadas con las fuentes tipográficas. Las más utilizadas son:

  • font-family: establece la familia a utilizar (arial, times new roman, courier, …) Existen dos valores por defecto: serif (tipografia con serifa) y sans-serif (tipografía sin serifa)
  • font-size: establece el tamaño de la fuente. Habitualmente en pixels, em o rem.
  • font-weight: establece el grosor de la fuente. Los valores habituales son: normal, bold, light.
  • font-style: nos permite indicar un texto en cursiva o normal con los valores “italic” y “normal”
  • text-align: define la alineación que queremos en el texto: left, right, center, justify
  • line-height: define la separación entre líneas, normalmente en px, em o rem.

Las fuentes tipográficas se suelen clasificar en 5 tipos básicos:

  • Serif
  • Sans-serif
  • Cursive
  • Fantasy
  • Monospace

Web safe fonts

En web, solo podemos ver correctamente una fuente si esta se encuentra instalada dentro de nuestro ordenador.

Si un diseñador web utiliza una fuente que no tenemos instalada, nuestro navegador intentará reemplazarla por otra provocando efectos no deseados.

Sólo hay un grupo de fuentes que están instaladas por defecto en todos los sistemas operativos tanto de desktop como mobile. Son las “Web Safe Fonts”

Además, si intentamos usar un grupo de fuente por defecto, es posible que tengamos diferentes resultados según el sistema operativo

h1 { font-family: sans-serif; font-size: 2em; font-weight: bold; font-style: italic; text-align: center; line-height: 1.4rem; }
  • Arial (sans-serif)
  • Verdana (sans-serif)
  • Tahoma (sans-serif)
  • Trebuchet MS (sans-serif)
  • Times New Roman (serif)
  • Georgia (serif)
  • Garamond (serif)
  • Courier New (monospace)
  • Brush Script MT (cursive)
  • Sans-serif
  • Serif
  • Monospace
  • Fantasy

font-family

Especificar una fuente tipográfica o un grupo de fuentes en CSS es sencillo. Se utiliza la propiedad ”font-family”.

Se pueden especificar varias fuentes, de manera que si no se encuentra la primera, intenta utilizar la siguiente.

Se suele poner un grupo de fuentes al final, por si no se ha encontrado ninguna de las fuentes anteriores.

h1 { font-family:serif; } p { font-family:Helvetica, Arial, sans-serif; }

Shadows, Border-radius, Text-Decoration

Shadows

Podemos aplicar una sombra a una caja con la propiedad box-shadow especificando:

  1. desplazamiento eje x
  2. desplazamiento eje y
  3. distancia de degradado
  4. color de sombra

Podemos también aplicar sombras a textos mediante la propiedad text-shadow especificando los mismos valores

code { box-shadow: 5px 5px 5px rgba(0,0,0,0.2) } h1 { text-shadow: 5px 5px 5px rgba(0,0,0,0.2) }

Border-radius

Podemos redondear las esquinas de nuestras cajas mediante la propiedad border-radius. Se puede especificar las unidades en px o en %, que se calcula a partir de la esquina.

Si indicamos que queremos un border-radius de 50%, nos aparecerá un recorte redondeado si la caja era cuadrada o oval si la caja era rectangular

img { border-radius: 50% }

Text decoration

Podemos decorar un texto de diferentes formas, por ejemplo subrallándolo o tachandolo

Podemos indicar también el estilo de la decoración, el color y el tamaño

span.removed { text-decoration: line-through; } a { text-decoration: none; } h2 { text-decoration: underline wavy blue 5px; }

Flexbox

Introducción

Flexbox es un método para organizar elementos en una dimensión, sea horizontal o vertical.

Para definir los elementos que queremos organizar, los debemos situar en un contenedor que tenga el display:flex

La propiedad flex-direction nos indica la dirección que deben situarse los elementos. row significa en una sola fila y column en una sola columna

Además podemos invertir el sentido con los valores row-reverse y column-reverse

Podemos indicar el espacio entre elementos mediante la propiedad gap

<div class="flexbox"> <div> <p>Element</div> </div> <div> <p>Element</div> </div> </div> .flexbox { display: flex; flex-direction: row; gap: 20px; }

Grids

Introducción

Grids CSS es un método para organizar elementos en dos dimensiones.

Para definir los elementos que queremos organizar, los debemos situar en un contenedor que tenga el display:grid

En un grid, debemos definir al menos las columnas que queremos y el ancho de éstas mediante la propedad grid-template-columns

Podemos especificar la situación de un elemento en una fila y una columna concreta mediante las propiedades grid-column y grid-row

Podemos indicar el espacio entre elementos mediante la propiedad gap

<div class="grid"> <div id="element1"> <p>Element</div> </div> <div id="element2> <p>Element</div> </div> </div> .grid { display: grid; grid-template-columns: 100px 100px; gap: 20px; } #element1 { grid-column:1; grid-row:2; }

Media queries

Las media queries nos permiten ejecutar declaraciones CSS sólo en ciertos tamaños de pantalla

Un consejo es plantear todo el CSS como si fuera una pantalla móvil y luego adaptar a pantallas mayores mediante media queries.

Son muy útiles para especificar el comportamiento de flexbox y grids en dispositivos móviles y dispositivos desktop

@media query and (min-width:768px) { h1 { color: red; } }

Display, position, overflow, z-index

Display

Especifica el comportamiento de un elemento. Los valores más comunes son:

  • block: establece que el elemento tiene que comportarse como un bloque. Valor por defecto para la mayoría de elementos (h1-h6, p, div, section, …)
  • inline: el elemento tiene que comportarse como un elemento en línea. Valor por defecto para elementos como <strong> , <em>, <a> , <span>, <img>, …</span>
  • flex: el elemenento debe usar flexbox
  • grid: el elemento usará grid CSS
  • inline-block: el elemento estará en línea, pero podremos aplicar propiedades reservadas al modelo de caja
  • none: el elemento no se visualizará en el navegador
img { display:block; } div.hidden { display:none; }

Aunque pensemos lo contrario, las imágenes se situan en línea con el resto de elementos. Es por eso que cuando queremos centrar una imagen en una pantalla tenemos muchos problemas. Aplicando que su display sea “block” y definiendo un “margin: auto”, haremos que nuestra imagen quede centrada en el documento.

Position

Establece el método de posicionamiento de elementos por pantalla. Por defecto será “static”, que significa que los elementos se situarán en el orden que aparece en el HTML y respetando el modelo de caja.

Otros valores son:

  • Absolute: relativo a su elemento padre
  • Fixed: relativo a la ventana del navegador
  • Sticky: basado en la posición del scroll
div#overlay { display: fixed; top:0; left:0; }

Overflow

Establece el comportamiento que debe suceder cuando los elementos rebasan el límite marcado por un contenedor. Sus valores pueden ser:

  • visible: los elementos serán visibles aunque rebasen el contenedor. Es el valor por defecto.
  • scroll: aunque el contenido no rebase el contenedor, aparecerá una barra de scroll
  • hidden: los contenidos estarán escondidos, pero mediante JS se puede realizar scroll
  • auto: si el contenido rebasa el contenedor aparecerá una barra de scroll
  • clip: el contenido estará escondido, y será inaccesible incluso con JS
div#mask { overflow: auto; }

z-index

Establece el orden de un elemento respeto el resto. Su valor es un numéro entero que puede ser positivo o negativo.

div#overlay { z-index: 999; }

Animaciones y transiciones

Animaciones

Existe la posibilidad de animar un elemento HTML definiendo los frames (pasos) que debe tener una animación. Para ello debemos utilizar la regla @keyframes indicando el nombre de la animación y definiendo las declaraciones CSS en cada uno de los pasos.

Una vez definida la regla @keyframes, debemos aplicar la animación a los elementos que deseemos definiendo las propiedades:

  • animation-name: el nombre de la animación a aplicar
  • animation-duration: los segundos que queremos que dure la animación

Existen otros parámetros para manipular las animaciones:

  • animation-delay: define el tiempo a esperar en segundos para ejecutar la animación
  • animation-iteration-count: define las veces que se va a ejecutar la animación (puede ser un número o infinito)
  • animation-direction: define como queremos que se comporte la animación. Puede ser normal (de 0% a 100%) reverse (de 100% a 0%), alternate (de 0% a 100% y luego a 0%) o alternate-reverse (de 100% a 0% y luego a 100%).
  • animation-timing-funcion: define la aceleración que queremos de la animación: linear, ease-in, ease-out,..
  • animation-fill-mode: define el estilo del elemento cuando la animación no se está ejecutando. Recomendación: poner el valor “both” para que se quede en la última posición.
@keyframes animationName { 0% {font-size: 16px;} 100% {font-size: 24px;} } div { animation-name: animationName; animation-duration: 2s; }

Transiciones

Las transiciones nos permiten cambiar una propiedad de forma suave, controlando su duración.

A diferencia de las animaciones CSS, no pueden crear bucles o movimimientos complejos, pero son muy útiles para animar elementos a partir de la interacción del usuario

Tenemos varias propiedades para controlar la transición:

  • transition-delay: define el tiempo a esperar en segundos para ejecutar la animación
  • transition-duration: define la duración de la transición
  • transition-property: nos permite definir diferentes valores según la propiedad que queremos aplicar la transición
  • transition-timing-funcion: define la aceleración que queremos de la animación: linear, ease-in, ease-out,..
div { width: 100%; transition: 2s; } div:hover { width: 300px; }
